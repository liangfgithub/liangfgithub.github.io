The Dishonest Casino¶

This example is taken from Durbin et. al. (1999): A dishonest casino uses two dice, one of them is fair the other is loaded.

  • State 0: The probabilities of the fair die are (1/6,…,1/6).
  • State 1: The probabilities of the loaded die are (1/10,…,1/10,1/2)

The observer doesn’t know which die is actually taken (the state is hidden), but the sequence of throws (observations) can be used to infer which die (state) was used.

In [2]:

gen_model.startprob_ = np.array([0.5, 0.5])
gen_model.transmat_ = np.array([[0.99, 0.01], [0.02, 0.98]])
gen_model.emissionprob_ = np.array([[1 / 6, 1 / 6, 1 / 6, 1 / 6, 1 / 6, 1 / 6],
                                    [1 / 10, 1 / 10, 1 / 10, 1 / 10, 1 / 10, 1 / 2]])

rolls, gen_states = gen_model.sample(2000)

Fit the data with two hidden states¶

Use Baum-Welch (i.e., EM) algorithm to estimate the model parameters.¶

Note that the algorithm would get stuck, if we initialize the emission distributions to be the same for each latent state.

Next we initilize the emission probability matrix $B$ as follows: generate positive entries for the $2\times 3$ matrix using a gamma distribution, and then divide each entry by the corresponding row sum to ensure each row is a probability vector summing to 1.

tmpB = np.random.default_rng(seed=100).gamma(shape=1, size=(2, 3))
tmpB = normalize(tmpB, axis=1, norm='l1')

myhmm2 = hmm.CategoricalHMM(n_components=2, algorithm='viterbi',
                            random_state=100, n_iter=1000, params='te', init_params='t')
myhmm2.startprob_ = np.array([0.5, 0.5])
myhmm2.emissionprob_ = tmpB

Predict Latent States using Individual Marginal Probabilities¶

Compute the (marginal) conditional distribution of the hidden state for each observation: $P(Z_i = k \mid X_1, \dotsc, X_n)$. Then classify $Z_i$ to be the most probable state.

In [11]:
mypost = myhmm2.predict_proba(observations)

Predict Latent States using the Viterbi algorithm¶

Use the Viterbi algorithm to compute the most probable hidden sequence.

Fit the data with three hidden states¶

The number of hidden states is a tuning parameter. We can try a range of hidden states and then use AIC/BIC to select $K$, the number of hidden states.

In [15]:
tmpB = np.random.default_rng(seed=100).gamma(shape=1, size=(3, 3))
tmpB = normalize(tmpB, axis=1, norm='l1')

Note that the likelihood of the data (integrated over all possible hidden sequences) could be computed using the score command.

In [19]:

Out[19]:

The two log-likelihoods are pretty close, so the model with two hidden states, which uses less number of parameters, will be chosen by AIC or BIC, once we add the penalty for the number of parametres.
